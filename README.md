警告： 
1.本项目首次尝试用js开发前后端大型网游，未经历上线考验。 

2.本项目程序、美术和策划均由我完成，故有些模块没时间完成。 

3.node.js由于太灵活不太适合做多人的大项目，对开发者的能力要求比较高。


优化： 
1.pomelo的消息传输机制太臃肿，应该自己定义一套网络传送机制，幸好js绑定C++模块很容易。 

2.高运算模块使用C++开发，由于时间有限，ai和寻路还是保持js的版本。 

3.尽量减少pomelo的服务器集群的规模。
